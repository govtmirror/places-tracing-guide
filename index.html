National Park Service

About Places

Places is an internal data collection system for the National Park Service's "core" geospatial data. We built Places so that all National Park Service employees, both technical and non-technical, can contribute their knowledge to the map.

Data from Places is fed directly into both Park Tiles and the larger National Park Service structured data initiative, which will be used to drive NPS.gov and a publicly available application programming interface (API) starting in January of 2016.

The Places database currently contains the following datasets (click each data type for a snapshot GeoJSON, updated from the database every five minutes):

  1. Points of Interest
  2. Buildings
  3. Roads
  4. Trails
  5. Parking Lots

These datasets were initially seeded from existing park, region, and program-level GIS datasets, but now anyone who is connected to the National Park Service network can edit them using the Places Editor (NPS-only). If your organization has more/better data available and you'd like to get it into Places, send us an email or create an issue on GitHub and we will assist with the upload.

Disclaimer: all data contributed to Places is public facing. Do not add any sensitive information unfit for public consumption. Only add data inside park boundaries!

The Places editor is built using OpenStreetMap's iD editor. Check out learnOSM's iD editor walkthrough to familiarize yourself with the interface. It is intentionally simple, developed with a non-technical audience in mind. So, fear not! Let's jump in and get started.

Note: must be logged into NPS network.

Points of Interest

Point feature

A point of interest or "POI" is represented by a point or "node" on the map and has geographic coordinates (latitude and longitude). A POI can be a standalone node or it can be part of a "way" (an ordered list of nodes representing a linear feature or area). Learn more about POIs from OpenStreetMap's wiki.

Places was originally named "Points of Interest" as POIs were the first (and only) feature type supported. Places supports a wide variety of POIs, a full list of which can be found in our data preset wiki. Don't see the tag you're looking for? Open an issue on GitHub.

This guide is intended to serve as a starter kit for contributing to Places, so we are going to review some of the most common/basic tags to add to the map. Use iD's Point tool to add a point to the map:

Ranger Station

A ranger station is a building or facility primarily intended to house park law enforcement and rescue staff, and is typically not open to the public. It can also be as simple as a ranger sitting at a campground kiosk. Ranger stations can be seasonal, and they are commonly confused with visitor centers as both can be located in the same building. In the event the two are co-located, the best practice is to create two coincident points - one for the ranger station and another for the visitor center. Learn more from the OSM wiki here, but keep in mind that definitions can vary between the Park Service and Forest Service, and between parks.

Tag a ranger station with a point on top of the building as amenity=ranger_station. Additionally, trace the building footprint and tag as building=yes (see the buildings section to learn more about tracing building footprints).

Buildings

Area feature

Buildings in Places are intentionally simple. We are primarily concerned with building footprints, while any additional building amenities are added as points of interest on top of the footprint. For example, add shop=general as a point of interest on top of the building footprint instead of building=retail as a tag included in the building footprint polygon to represent a general store (see store above).

Trace a building by using the Area tool to trace the building footprint and tag as building=yes.

Note: Don't get too carried away with nodes while tracing - keep it simple to conform to the general shape of the building visible from the imagery. More nodes mean more data, which can ultimately slow the database.

Additionally, if you know the location of the building's entrance or exit you can add it as a vertex on the building footprint polygon in the appropriate location. Tag a building entrance/exit as building=entrance. Learn more from the OSM wiki here.

Bridge

A bridge is a structure used to carry a road, railway, or similar feature over another feature. Learn more from the OSM wiki here.

Tag a bridge by splitting the upper way at each end of the bridge (where it joins solid ground) and add the tag bridge=yes (remember to maintain the road's existing classification, e.g. highway=primary). Also, add layer=1, which will create the hierarchy indicating the upper way is above the lower.

Feel free to add additional information using the following keys:

  • name=
  • operator=
  • addr=

Tunnel

A tunnel is an underground or underwater passageway enclosed except for its entrance and exit at each end. Learn more from the OSM wiki here.

Tag a tunnel by splitting the enclosed section of the way and adding the tag tunnel=yes (remember to maintain the road's existing classification, e.g. highway=primary). Also, add layer=-1, which will create the hierarchy indicating the tunnel is underneath the feature it intersects.

Feel free to add additional information using the following keys:

Tag a sidewalk road by tracing its centerline from beginning to end as highway=footway and footway=sidewalk. If the sidewalk crosses a road, tag the associated way with footway=crossing.

Intersections

An intersection occurs where two or more roads meet. To create a simple intersection between two roads, remember to connect them with a node at the point of intersection.

Surface

Surface describes the physical surface of a feature (in this case, a road or trail). Learn more from the OSM wiki here.

Tag a feature's surface by adding the surface= tag to its associated way. The following surface types are mapped directly to the current EGIS standards (click each for the wiki, which includes other surface types as well):

  • surface=asphalt
  • surface=concrete
  • surface=paving_stones
  • surface=cobblestone
  • surface=gravel
  • surface=sand
  • surface=ground
  • surface=paved
  • surface=unpaved

Oneway

Oneway indicates the oneway access restriction on roads. Learn more from the OSM wiki here.

Tag a road with oneway access as oneway=yes to its associated way. Note that ways have a directionality, so the direction in which you map the way determines its directionality. However, you can tag a way with oneway=-1 to reverse its existing directionality (appropriate when a way's directionality already governs another attribute such as a bus route).

Lanes

Lanes indicate the number of traffic lanes on a highway that are available to motorised traffic (bicycle lanes are therefore excluded). Learn more from the OSM wiki here.

Tag a road with lanes= (e.g. lanes=2) to indicate how many lanes it has.

Maxspeed

Maxspeed indicates the maximum legal speed limit for general traffic on a road. Learn more from the OSM wiki here.

Tag a road with maxspeed= (e.g. maxspeed=45 mph) to indicate the speed limit, including the unit of measure (mph for miles per hour). Places defaults to miles per hour, so maxspeed=45 is effectively the same as maxspeed=45 mph. Best practice is to include units, however.

Piste

A piste is a trail intended for winter sports such as downhill and cross-country skiing, dog sledding, and snowshoeing. Learn more from the OSM wiki here.

Tag a piste by tracing its centerline from beginning to end using the piste:type= key along with its appropriate value (click for link to OSM wiki):

  • Backcountry Ski Trail (piste:type=skitour)

For example, if a trail is both a generic footpath (intended for hiking) and a downhill ski trail tag the associated way highway=path and piste:type=downhill.

Water Trail

A water trail is a trail running through a waterway, intended for canoes, kayaks, motorboats, etc. Tag a water trail by tracing its centerline from beginning to end as waterway=yes along with its access conditions, as appropriate (click for link to OSM wiki):

  • Motorized Watercraft Trail (motorboat=) for a route accessible to motorized watercraft such as a powerboat or motorized sailboat.

The start and end of a water trail is often a boat launch, which is added as a point of interest. Click here to learn more about tagging boat launches in Places.

Parking

Area, point, and line feature

Parking takes two primary forms in Places: parking (a generic term, indicating parking is available as an amenity, typically in the form of a parking lot or garage) and parking aisles. Learn more from the amenity=parking and service=parking_aisle OSM wikis.

Parking types (click title to jump to type):

  • Parking: amenity=parking
  • Parking Aisle: highway=service and service=parking_aisle

Parking (amenity)

Parking is a facility available for use by the public, customers, or other authorized users for the parking of motor vehicles (cars, trucks, motorcycles, etc). This can come in various forms, including a parking lot or garage, among others. Learn more from the OSM wiki here.

Best practice is to trace a parking lot's footprint with the Area tool and add a node in the center of the area with the Point tool and tag both as amenity=parking.

If the parking area is undefined simply add the point without defining an area. If the feature is a parking garage or building, add the tag building=yes to the polygon.

Feel free to add additional information using the following keys:

  • name=
  • operator=
  • addr=
  • capacity=
  • maxstay=
  • fee=
  • access=

Parking Aisle

A parking aisle is a subordinated way in a parking lot between rows of parking spaces that vehicles use to drive into and out of the spaces. Learn more from OSM wiki here.

Tag a parking aisle by tracing it as you would a service road using the Line tool and tag as highway=service and service=parking_aisle.

The main way(s) in the parking lot, for entering and connecting multiple parking aisles, should be tagged highway=service only.

Feel free to add additional information using the following keys:

  • name=
  • surface=
  • oneway=
  • access=
