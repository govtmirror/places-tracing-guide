Places Tracing Guide

National Park Service

About Places

Places is an internal crowd-sourcing system for the National Park Service's "core" geospatial data. We built Places so that all National Park Service employees, both technical and non-technical, can contribute their knowledge to "the map".

Data from Places is fed directly into both Park Tiles and the larger National Park Service structured data initiative, which will be used to drive NPS.gov and a publicly available application programming interface (or API) starting in January of 2016.

The Places database currently contains the following datasets (click each data type for a snapshot GeoJSON, updated from the database every five minutes):

  1. Points of Interest
  2. Buildings
  3. Roads
  4. Trails
  5. Parking Lots

These datasets were initially seeded from existing park, region, and program-level GIS datasets, but now anyone who is connected to the National Park Service network can edit them using the Places Editor (NPS-only). If your organization has more/better data available and you'd like to get it into Places, send us an email or create an issue on GitHub and we will assist with the upload.

The Places editor is built using OpenStreetMap's iD editor. Check out learnOSM's iD editor walthrough to familiarize yourself with the interface. It is intentionally simple, developed with a non-technical audience in mind. So, fear not! Let's jump in and get started.

Points of Interest

Point feature

A point of interest or "POI" is represented by a point or "node" on the map and has geographic coordinates (latitude and longitude). A POI can be a standalone node or it can be part of a "way" (an ordered list of nodes representing a linear feature or area). Learn more about POIs from OpenStreetMap's wiki.

Places was originally named "Points of Interest" as POIs were the first (and only) feature type supported. Places supports a wide variety of POIs, a full list of which can be found in our data preset wiki. Don't see the tag you're looking for? Open an issue on GitHub.

This guide is intended to serve as a starter kit for contributing to Places, so we are going to review some of the most common/basic tags to add to the map. Use iD's Point tool to add a point to the map:

Campground

A campground is an area, usually divided into a number of pitches (or "campsites," see below), where people can camp overnight using tents or camper vans or caravans. Learn more from the OSM wiki here.

Tag a campground with a point in the middle of the area as tourism=camp_site. Feel free to add additional information using the following keys (click for link to OSM wiki):

  • name=
  • operator=
  • addr=
  • opening_hours=
  • capacity=
  • fee=

Ranger Station

A ranger station is a building or facility primarily intended to house park law enforcement and rescue staff, and is typically not open to the public. It can also be as simple as a ranger sitting at a campground kiosk. Ranger stations can be seasonal, and they are commonly confused with visitor centers as both can be located in the same building. In the event the two are co-located, the best practice is to create two coincident points - one for the ranger station and another for the visitor center. Learn more from the OSM wiki here, but keep in mind that definitions can vary between the Park Service and Forest Service, and between parks.

Tag a ranger station with a point on top of the building as amenity=ranger_station and building=yes. Additionally, trace the building footprint and tag as building=yes (see the buildings section to learn more about tracing building footprints). Feel free to add additional information using the following keys (click for link to OSM wiki):

Buildings

Area feature

Buildings in Places are intentionally simple. We are only concerned with building footprints, while any additional building amenities are added as points of interest on top of the footprint. For example, add shop=general and building=yes as a point of interest on top of the building footprint instead of building=retail as a tag included in the building footprint polygon to represent a general store (see store above).

Trace a building by using the Area tool to trace the building footprint and tag as building=yes.

Don't get too carried away with nodes while tracing - keep it simple to conform to the general shape of the building visible from the imagery.

Roads

Linear feature

Roads and trails are identified primarily by the highway= field (click for link to OSM wiki). Roads are distinct from trails in that they are generally intended for use by motor vehicles, while trails are generally intended for recreation such as hiking, biking, horseback riding, skiing, etc.

There are a few exceptions to this rule, including all-terrain vehicle (ATV) trails, four-wheel drive (4wd) trails, motorized watercraft trails, and snowmobile trails, which all allow their respective type of motor vehicle access.

Trace a road by using the Line tool to trace its centerline from beginning to end:

The tags for different road types will vary, classified below. If the road classification is unknown, default to the highway=unclassified tag (click to jump to description).

Click title to jump to road type:

  • Primary Road: highway=primary
  • Secondary Road: highway=secondary
  • Tertiary Road: highway=tertiary
  • Residential Road: highway=residential
  • Unclassified Road: highway=unclassified
  • Service Road: highway=service
  • Bridge: bridge=yes
  • Sidewalk: highway=footway and footway=sidewalk

Dealing with intersecting roads? Click here to make sure you are connecting them properly.

Primary Road

A primary road is a major highway, normally with two lanes. Learn more from the OSM wiki here.

Bridge

A bridge is a structure used to carry a road, railway, or similar feature over another feature. Learn more from the OSM wiki here.

Tag a bridge by splitting the upper way at each end of the bridge (where it joins solid ground) and tag as bridge=yes. Also, add layer=1, which will create the hierarchy indicating the upper way is above the lower.

  • name=
  • operator=
  • addr=

Sidewalk

A sidewalk is a part of a road dedicated for pedestrian use, usually separated from the road by a curb. Learn more from the OSM wiki here.

Tag a sidewalk road by tracing its centerline from beginning to end as highway=footway and footway=sidewalk. If the sidewalk crosses a road, tag the associated way with footway=crossing. Feel free to add additional information using the following keys:

  • name=
  • operator=
  • surface=
  • lit=
  • width=
  • access=

Intersections

An intersection occurs where two or more roads meet. To create a simple intersection between two roads, remember to connect them with a node at the point of intersection.

Surface

Surface describes the physical surface of a feature (in this case, a road or trail). Learn more from the OSM wiki here.

Tag a feature's surface by adding the surface= tag to its associated way. The following surface types are mapped directly to the current EGIS standards (check out the wiki for other common tags):

  • surface=asphalt
  • surface=concrete
  • surface=paving_stones
  • surface=cobblestone
  • surface=gravel
  • surface=sand
  • surface=ground
  • surface=paved
  • surface=unpaved

Oneway

Oneway indicates the oneway access restriction on roads. Learn more from the OSM wiki here.

Tag a road with oneway access as oneway=yes to its associated way. Note that ways have a directionality, so the direction in which you map the way determines its directionality. However, you can tag a way with oneway=-1 to reverse its existing directionality (appropriate when a way's directionality already governs another attribute such as a bus route).

Lanes

Lanes indicate the number of traffic lanes on a highway that are available to motorised traffic (bicycle lanes are therefore excluded). Learn more from the OSM wiki here.

Tag a road with lanes= (e.g. lanes=2) to indicate how many lanes it has.

Max speed

Maxspeed indicates the maximum legal speed limit for general traffic on a road. Learn more from the OSM wiki here.

Tag a road with maxspeed= (e.g. maxspeed=45 mph) to indicate the speed limit, including the unit of measure (mph for miles per hour). Places defaults to miles per hour, so maxspeed=45 is effectively the same as maxspeed=45 mph. Best practice is to include units, however.

Trails

Linear feature

Like roads, trails are identified primarily by the highway= field (click for link to OSM wiki). Trails are distinct from roads in that they are generally intended for recreation, while roads are generally intended for use by motor vehicles. As noted in the roads section, the exceptions to this rule are: all-terrain vehicle (ATV) trails, four-wheel drive (4wd) trails, motorized watercraft trails, and snowmobile trails, which all allow their respective type of motor vehicle access.

Trace a trail by using the Line tool to trace its centerline from beginning to end:

The tags for different trail types will vary, classified below. If the trail classification is unknown, or if it is only accessible on foot, default to the highway=path tag (click to jump to description). If a trail is accessible to more than just walking/hiking (e.g. horse=yes for a trail accessible to equestrians), default to the highway=track tag and include the associated accessibility.

Note: remember to tag the start of each trail with highway=trailhead (click to jump to feature description).

Click title to jump to generalized trail type (multi-use and water trails require additional tags to classify properly):

  • Trail (footpath): highway=path
  • Multi-use trail: highway=track
  • Water trail: waterway=yes

Trail (footpath)

A trail, also commonly referred to as a footpath, is a trail intended for pedestrian use or "foot traffic" (typically intended for hiking, in the NPS context). Learn more from the OSM wiki here.

Tag a trail by tracing its centerline from beginning to end as highway=path. Feel free to add additional information using the following keys:

  • name=
  • operator=
  • tracktype=
  • surface=
  • width=
  • incline=
  • mtb:scale=
  • mtb:scale:uphill=
  • mtb:scale:imba=
  • sac_scale=
  • trail_visibility=
  • access=

Multi-use trail

A "multi-use" trail is not a specific trail type, rather it is a generalized term used to group the variety of trail types accessible to more than just pedestrians. The default tag for a "multi-use" trail starts with highway=track and includes its access conditions (e.g. horse=yes for equestrian access, bicycle=yes for bicycle access, etc.). The various access conditions are as follows (click for link to OSM wiki):

  • ATV trail (atv=)
  • Bicycle trail (bicycle=)
  • Dog sled trail (piste:type=sleigh)
  • Equestrian trail (horse=)
  • Four-wheel drive trail (4wd_only=)
  • Motorcycle trail (motorcycle=)
  • Snowmobile trail (snowmobile=)
  • Downhill ski trail (piste:type=downhill)
  • Cross-country ski trail (piste:type=nordic)
  • Snowshoe trail (piste:type=hike)

Tag a trail by tracing its centerline from beginning to end as highway=path. Feel free to add additional information using the following keys:

  • name=
  • operator=
  • tracktype=
  • surface=
  • width=
  • incline=
  • mtb:scale=
  • mtb:scale:uphill=
  • mtb:scale:imba=
  • sac_scale=
  • trail_visibility=
  • access=

Water trail

A "water" trail is not a specific trail type, rather it is a generalized term used to group the variety of water-based trail types. The default tag for a "water" trail starts with waterway=yes and includes its access conditions, depending on what type of watercraft is permissable. Places supports the following two water-based access conditions (click for link to OSM wiki):

  • Non-motorized watercraft trail (canoe=) for a route accessible to non-motorized watercraft such as a canoe or kayak.
  • Motorized watercraft trail (motorboat=) for a route accessible to motorized watercraft such as a powerboat or motorized sailboat.

Tag a water-based trail by tracing its centerline from beginning to end as highway=path. Feel free to add additional information using the following keys:

  • name=
  • operator=
  • whitewater=
  • width=
  • section_grade=
